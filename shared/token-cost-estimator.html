Four things to take away

Tokens, not words

A token is a chunk of text — roughly ¾ of a word on average. Usage and price are counted in tokens, so a "word count" understates it.

You pay twice

You're charged for input (what you send) and output (what it writes back) — and output usually costs several times more per token.

Most single requests are cents

One request is usually tiny. Cost adds up through volume — running the same task hundreds of times a week.

Trim the input

Sending a whole 30-page document when a 1-page summary would do wastes tokens every time. Trim first; the saving repeats.

Mini glossary

Token
The chunk of text AI works in — a word, part of a word, or punctuation. About ¾ of a word on average.
Input tokens
Everything you send: your prompt plus any document or context attached to it.
Output tokens
Everything the AI writes back. Usually priced higher per token than input.
Context window
The maximum tokens a model can consider at once — input and output combined.
$ per 1M tokens
How API pricing is quoted. Divide by a million to get the cost of a single token.
Throughput
How fast a model produces tokens — it drives how long a reply takes.
